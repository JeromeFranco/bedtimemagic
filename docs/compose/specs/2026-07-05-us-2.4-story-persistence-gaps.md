# US-2.4 Story Persistence — Gap Fill Spec

**Date:** 2026-07-05
**Status:** Approved
**Source:** `docs/user-stories.md` US-2.4

---

## [S1] Problem

Story persistence is largely implemented (DB schema, edge function persistence, API functions), but several gaps remain:

1. No React Query hooks for story data fetching
2. Story screen receives full story JSON via route params instead of fetching by ID
3. No UPDATE RLS policy on `stories` table
4. `getStories` and `logStoryRating` API functions exist but are unused

## [S2] Solution Overview

- Add React Query hooks wrapping existing API functions
- Refactor story screen to fetch by ID using the new hook
- Add UPDATE RLS policy via new migration
- Wire up callers to pass story ID instead of JSON

## [S3] React Query Hooks

**File:** `src/hooks/use-story.ts`

Two hooks wrapping existing API functions from `src/api/stories.ts`:

- `useStory(id: string)` — `useQuery` wrapping `getStory(id)`, key: `['story', id]`, `staleTime: 5min`
- `useStories(childId?: string)` — `useQuery` wrapping `getStories(childId)`, key: `['stories', childId]`, `staleTime: 5min`

Both use the existing `QueryClient` from `src/lib/query-client.ts` (already configured with `staleTime: 5min`, `retry: 1`).

## [S4] Story Screen Refactor

**File:** `src/app/story.tsx`

Current: receives `story` as `JSON.stringify`'d route param, parses inline.
New: receives `id` as route param, uses `useStory(id)` hook.

Changes:
- `useLocalSearchParams<{ id: string }>()` instead of `{ story: string }`
- `const { data: story, isLoading, error } = useStory(id)`
- Loading state: reuse existing calm aesthetic (could show a simple loading indicator)
- Error state: existing error UI with "Go Back" button
- Remove `JSON.parse` try/catch block

**Callers to update:**
- `src/app/generate.tsx` — after story generation, navigate with `story.id` instead of `JSON.stringify(story)`
- Any other screen that navigates to `/story`

## [S5] UPDATE RLS Policy

**New migration:** `supabase/migrations/20260705000000_add_story_update_policy.sql`

```sql
create policy "Users can update their own stories"
  on public.stories for update
  to authenticated
  using ((select auth.uid()) = user_id)
  with check ((select auth.uid()) = user_id);
```

Matches the pattern of existing INSERT/SELECT/DELETE policies. Uses `(select auth.uid())` subquery form per existing convention.
